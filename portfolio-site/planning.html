Planning & Architecture

Designed with enterprise patterns and scalability in mind. Clean Architecture ensures separation of concerns while CQRS provides optimal read/write optimization.

Key Architecture Decisions

  • check_circle
    CQRS with MediatR

    Separates read and write operations for better scalability and maintainability

  • check_circle
    Repository Pattern

    Abstracts data access, making it easy to switch databases or add caching

  • check_circle
    Dependency Injection

    All dependencies are injected, enabling testability and loose coupling

  • check_circle
    Domain-Driven Design

    Rich domain models with business logic encapsulated in entities

Domain Layer

Contains business entities (Student, Course, Enrollment, etc.), domain events, and business rules. No external dependencies.

Application Layer

Contains CQRS commands/queries, validation logic (FluentValidation), and application services. Depends only on Domain.

Infrastructure Layer

Implements data access with EF Core, external service integrations, and cross-cutting concerns like logging.

Presentation Layer

ASP.NET Core Web API with controllers, middleware, and configuration. Angular frontend with lazy-loaded modules.

Core Data Model

person

Student

  • StudentId, Name, Email
  • DateOfBirth, Gender
  • ProgramId, DepartmentId
  • Status, Type, Standing
  • GPA, TotalCredits
  • Has Many: Enrollments, Documents
menu_book

Course

  • Code, Name, Description
  • CreditHours, Level
  • DepartmentId
  • Prerequisites
  • IsActive
  • Has Many: Sections
assignment

Enrollment

  • StudentId, SectionId
  • EnrollmentDate
  • Status (Enrolled, Dropped...)
  • Grade, GradePoints
  • AttendancePercentage
  • Belongs To: Student, Section
schedule

CourseSection

  • CourseId, TermId
  • SectionNumber
  • InstructorId
  • RoomId, Schedule
  • Capacity, EnrolledCount
  • Has Many: Enrollments
payments

BillingAccount

  • StudentId
  • Balance, Status
  • FinancialHold
  • Has Many: Charges, Payments
domain

Department

  • Code, Name
  • Description
  • IsActive
  • Has Many: Programs, Courses

Security Architecture

vpn_key Authentication

  • check JWT Bearer tokens with configurable expiration
  • check Refresh token rotation for extended sessions
  • check ASP.NET Core Identity for user management
  • check Password hashing with BCrypt
  • check Account lockout after failed attempts

shield Authorization

  • check Role-based access control (RBAC) with 6 roles
  • check Permission-based authorization for fine-grained control
  • check Resource-based auth (users access own data only)
  • check Policy-based authorization in controllers
  • check Route guards in Angular for frontend protection
